# Postmortems & RCAs — failure analysis

Retrospective analyses of something that went **wrong**: postmortems of
specific incidents/bugs, and root-cause analyses (RCAs) of systemic discipline
lapses. The two are **siblings** — co-equal kinds of failure analysis — and
share this directory. Distinct from project/phase/release retrospectives, which
live in `../retrospective/` (those look back on a *completed cycle*, not a
failure).

Point-in-time records: per ADR-0005's archive-style convention they are not
retro-edited to track later moves; their cross-references reflect the state at
authoring time.
