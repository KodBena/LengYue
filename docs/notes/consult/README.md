# Consult records

Verbatim records of analytic-firewall consultations — independent Opus-agent
reviews commissioned to pressure-test a plan or decision before it's
committed. Extracted from the flat `docs/notes/` root as their own taxon
(ADR-0005 Rule 5).

**Convention** (the standing SOP):

- **Filename:** `opus-consult-<date>-<topic>.md` (e.g.
  `opus-consult-2026-06-02-work-status-ssot-plan.md`).
- **Body:** a short framing (what was asked, what was decided in response),
  then the agent's response **verbatim**, then an `## Appendix — verbatim
  prompt` reproducing the prompt the agent was given, then a
  `License: Public Domain (The Unlicense)` line.
- Consult records are point-in-time artifacts. Per ADR-0005's archive-style
  convention they are **not** retro-edited to track later moves; their
  cross-references reflect the state at consultation time.

A consult record is **not** a design note: it captures an external review,
not our own planning. Design notes live in `docs/notes/design/`.
