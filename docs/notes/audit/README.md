# Audits — discrete audit reports

Point-in-time audit reports: codebase sweeps (e.g. stringly-typed-contracts),
liveness audits, and performance audits. Per ADR-0005's archive-style
convention they are not retro-edited; their cross-references reflect the state
at audit time.

Two audit-*adjacent* docs are deliberately **not** here — they are a different
genre (running notes / reflections, not reports) and stay in `docs/notes/`
root: `auditor-notes.md` (the auditor's perpetual working log) and
`audit-reflections.md` (reflections on the auditing practice, the companion to
`reflection.md`).
