# Retrospectives — looking back on a completed cycle

Release, phase, and research retrospectives — reflections on a *completed*
cycle or experiment (a shipped release, a training phase, an optimizer run).
Distinct from failure analysis (`../postmortem/`), which dissects something
that went wrong.

Point-in-time records: per ADR-0005's archive-style convention they are not
retro-edited; their cross-references reflect the state at authoring time.
