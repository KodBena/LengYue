# RFCs

Proposals for project-level changes that warrant discussion before
adoption. RFCs are distinct from sibling genres:

- **ADRs** (`docs/adr/`) — decisions that have been made.
- **Notes** (`docs/notes/`) — descriptive design documents about
  existing or planned work.
- **Playbooks** (`docs/playbooks/`) — procedural sequences for
  performing a known task.
- **Dispatches** (`docs/dispatch/`) — cross-team communications.

An RFC describes a proposed change, the alternatives considered,
the open questions, and the acceptance criteria. When accepted, an
RFC's outcome typically lands as an ADR (if the change is a
decision), a playbook (if procedural), a TODO entry (if scheduled
work), or a code change (if direct).

RFCs are numbered sequentially; the number is permanent regardless
of acceptance status. Per ADR-0005's authoring posture, an RFC's
filename uses the convention `NNNN-short-handle.md`, mirroring the
ADR convention.
