## Approach

### The cluster

Lines 152–177 of `App.vue` formed a closed unit:

- `isResizing` ref (152) — internal closure flag.
- `lastMouseX` plain `let` (153) — internal closure state, never
  templated.
- `startResize(e)` (155–162) — public entry; bound at
  `App.vue:342` via `@mousedown="startResize"`.
- `onMouseMove(e)` (164–170) — private; mutates
  `store.session.ui.controlPanelWidth` clamped to [200, 800].
- `stopResize()` (172–177) — private; symmetric cleanup.

External effects: `mousemove`/`mouseup` document listeners; a
`resizing` class on `document.body` driving a global CSS cursor
change. Grep confirmed zero references to cluster identifiers
outside lines 152–177 except `startResize` at line 342.

### Edits

**New file: `frontend/src/composables/useResizablePanel.ts`** —
46 lines including ADR-0006 header. Public API:
`useResizablePanel(): { startResize: (e: MouseEvent) => void }`.
Body is the verbatim cluster, with `store` imported from
`../store`. Behavior preservation is exact — same clamp, same
body-class toggle, same plain-`let` closure for `lastMouseX`. No
`onUnmounted` cleanup added (see Out of scope).

**Edits to `frontend/src/App.vue`** — three small changes:

1. Add `import { useResizablePanel } from './composables/useResizablePanel';`
   alongside the existing composable imports.
2. Remove lines 152–177 (the cluster).
3. Replace with one setup line:
   `const { startResize } = useResizablePanel();`

No template change — `App.vue:342` continues to bind
`@mousedown="startResize"` because the destructured name matches.

Convention alignment confirmed by reading `useScopedScroll.ts`
(effectful-DOM sibling) and `useAuth.ts` (most recent composable);
the new file follows the latter's ADR-0006 header style and
App.vue's semicolon style (the cluster's source).
