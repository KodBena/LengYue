Note: the boundary in the 0.020–0.030s strip is non-deterministic.
Two earlier Node runs and this Python run produce three different
specific "flip" patterns in that range. Sample evidence (firstReport
fast / cadence per run, same fixed-2.16s cadence):

  firstReport   Node run #1   Node run #2   Python run
  0.018         NO-PACKET*    NO-PACKET*    cadence
  0.020         cadence       cadence       fast
  0.022         cadence       fast          cadence
  0.025         fast          fast          fast
  0.028         cadence       fast          fast
  0.030         fast          cadence       fast

*The two NO-PACKET cells at firstReport=0.018 are a KataProxy-side
artefact, not a KataGo one — those Node runs were against KataProxy's
SELECTOR-role proxy, which has adaptive_reevaluate middleware engaged.
The transparent-proxy Python run shows the same firstReport=0.018 value
landing on the cadence pin (2207ms) cleanly. So the NO-PACKET shape is
proxy-side, while the noisy fast/cadence flip in the 0.020–0.030 strip
is KataGo-side.
